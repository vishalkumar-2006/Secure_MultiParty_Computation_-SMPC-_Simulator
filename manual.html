Secure Multiparty Computation Simulator

Overview

The Secure Multiparty Computation (SMPC) Simulator is an interactive, educational tool that demonstrates how secrets can be securely distributed and reconstructed using cryptographic protocols. It supports both:

  • Additive Secret Sharing - simple modular arithmetic for share distribution.
  • Shamir Secret Sharing - polynomial-based scheme using Lagrange interpolation.

The simulator shows how each share is generated and how the original secret can be reconstructed only when enough parties collaborate. It is designed for students, educators, and researchers studying secure distributed computation, privacy-preserving algorithms, and modular arithmetic.

Mathematical Background

1. Additive Secret Sharing

Given a secret S and modulus p (prime), the secret is divided among n parties as follows: 

Choose n-1 random numbers: r₁, r₂, ..., rₙ₋₁ ∈ [0, p)
Compute last share: rₙ = (S - Σᵢrᵢ) mod p
Then: S ≡ (r₁ + r₂ + ... + rₙ) mod p

Any party alone learns nothing about S. The secret can be reconstructed only if all shares are combined.

2. Shamir Secret Sharing

The secret S is encoded as the constant term of a random polynomial f(x) of degree t-1: 

f(x) = a₀ + a₁x + a₂x² + ... + aₜ₋₁x^(t-1)  (mod p)
where a₀ = S
Each share is a point (xᵢ, f(xᵢ))

Any t shares can reconstruct the secret using Lagrange interpolation: 

S = f(0) = Σ [ yᵢ * Π (0 - xⱼ)/(xᵢ - xⱼ) ] mod p
           (for all j ≠ i)

This scheme is t-out-of-n threshold secure, meaning fewer than t shares reveal nothing about S.

Behind the Scenes (Algorithms)

  • Modular Arithmetic: All calculations are performed under a prime modulus to prevent overflow and ensure reversibility using modular inverses.
  • Randomness: Shares are generated using uniform random integers modulo p.
  • Exponentiation: Modular exponentiation (modPow) is used for computing inverses efficiently via Fermat's Little Theorem.
  • Reconstruction: Each reconstruction step is displayed numerically, showing numerator, denominator, and modular inverse operations for transparency.

How to Run the Simulation

  1. Open the Simulator: Load SMPC.html in your browser.
  2. Select Protocol: Choose “Additive Sharing” or “Shamir Secret Sharing”.
  3. Configure Parameters:
    • Enter the secret (an integer < modulus).
    • Set the number of parties n.
    • For Shamir’s scheme, specify threshold t.
    • Choose a prime modulus (a number greater than the secret).
  4. Run Simulation: Press Run Next Step (S) to simulate share generation and reconstruction.
  5. Observe Output:
    • The Simulation Steps panel shows modular computations step-by-step.
    • The Protocol Flow panel visualizes message passing and progress.
    • The progress bar indicates overall completion.
  6. Reset: Press Reset (R) to clear data and start again.

Example Walkthrough

Additive Sharing Example
Secret S = 123, n = 3, p = 257
Random shares: 50, 70
Last share: (123 - (50+70)) mod 257 = 260 mod 257 = 3
Shares: [50, 70, 3]
Reconstruction: (50 + 70 + 3) mod 257 = 123
Shamir Sharing Example
Secret S = 42, t = 2, n = 3, p = 97
Polynomial: f(x) = 42 + 15x (mod 97)
Shares: (1,57), (2,72), (3,87)
Using any 2 shares:
f(0) = y₁*(0-x₂)/(x₁-x₂) + y₂*(0-x₁)/(x₂-x₁) = 42 (mod 97)

Keyboard Shortcuts

  • S - Run next simulation step
  • R - Reset simulation
  • H - Scroll to top or show help

Troubleshooting & Notes

  • Invalid modulus: Ensure modulus is prime and larger than the secret.
  • Incorrect threshold: Threshold t must be ≤ number of parties n.
  • Empty inputs: Fill all fields before running simulation.
  • Browser support: Works best on latest Chrome, Firefox, or Edge.
  • Educational mode: The simulator is designed for clarity, not for cryptographic production use.
